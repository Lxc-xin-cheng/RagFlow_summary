### 3. 专业术语与重难点名词解析
| 中文术语 | 英文对照 | 408核心考点/易错提示 |
|----------|----------|----------------------|
| 时间局部性 | Temporal Locality | 支撑Cache保留近期数据、TLB缓存、页缓存；**非**指“数据存放时间长” |
| 空间局部性 | Spatial Locality | 支撑Cache块预取、按行优先访问数组；**注意**：块大小与数组步长的匹配决定命中率 |
| 堆栈类算法 | Stack Algorithm | LRU/OPT/NRU属于此类，具有**包含性**，**不会出现Belady异常** |
| 队列类算法 | Queue Algorithm | FIFO属于此类，**可能出现Belady异常**；仅依赖调入时间，忽略访问频率 |
| 冗余ACK | Redundant ACK | 连续3个相同ACK触发快重传；**非**拥塞信号，而是“后续报文已到”的轻度过载提示 |
| 拥塞窗口 | Congestion Window (cwnd) | 快恢复时 `cwnd ← ssthresh`（非1）；超时后 `cwnd ← 1`；**常与rwnd混淆**，实际发送窗口=`min(cwnd, rwnd)` |
| 片偏移 | Fragment Offset | 单位为**8字节**（非1字节）；除最后一片外，数据长度必为8B整数倍 |
| 首部检验和 | Header Checksum | **仅校验首部**，不覆盖数据；每经路由器因TTL/标志变化需**重新计算** |
| 二进制指数退避 | Binary Exponential Backoff | 第k次冲突退避区间 `[0, 2^min(k,10)-1] × τ`；最大重传16次超限丢弃 |

> 🔍 **术语使用建议**：408答题时务必使用标准术语（如“堆栈类算法”“包含性”“乘法减小”），避免口语化表述；中英对照有助于快速定位教材原话与真题解析。

### 4. 知识薄弱点与高频丢分预警
| 模块 | 理解漏洞/细节盲区 | 408高频丢分点 | 针对性突破建议 |
|------|------------------|----------------|----------------|
| **数据结构** | 链表插入/删除O(1)的前提是“已知操作位置”，定位仍需O(n)；未意识到静态链表用数组下标模拟指针 | 选择题常考“已知第i个结点插入的时间复杂度”（答O(1)错，应为O(n)） | 牢记“定位O(n)+修改O(1)=整体O(n)”；动态分区空闲链表粒度细→管理开销大但操作简单，粒度粗→效率高但需处理合并 |
| **计组** | 局部性原理与具体硬件机制的对应关系模糊；未掌握Cache块大小与数组访问模式的定量关系 | 计算题：给二维数组访问代码+Cache参数，求命中率；易错在“片偏移单位”“块内地址计算” | 建立映射：时间局部性→Cache保留/TLB/页缓存；空间局部性→块预取/按行优先；动手算1道Cache命中率真题 |
| **操作系统** | LRU硬件开销大，实际多用Clock算法近似；缺页中断处理中“M=1写回”与“重新执行指令”的因果逻辑不清 | 排序题：缺页中断处理步骤排序（常错在“更新页表”与“恢复执行”的顺序）；Belady异常归属判断 | 画缺页中断流程图，标出P位/M位判断节点；明确LRU是“理论最优可实现算法”，OPT不可实现 |
| **计网** | TCP四算法窗口变化规则易混；CSMA/CD最短帧长计算未掌握τ与速率的反比关系；IPv4分片重组条件不清 | 计算题：给页面走向/TCP初始cwnd，求缺页率/窗口变化；IPv4首部检验和覆盖范围判断 | 制作TCP窗口变化速查表（超时→cwnd=1/ssthresh=cwnd/2；3冗余ACK→cwnd=ssthresh/ssthresh=cwnd/2）；最短帧长=`2τ×速率`，速率↑则τ必须↓ |

> ⚠️ **共性盲区**：你的提问偏向“机制理解”，但408真题中这些机制几乎全部以**计算推导、边界条件判断、异常场景分析**形式出现。例如：
> - TCP快恢复的 `cwnd` 变化是 `cwnd ← ssthresh`，但 `ssthresh` 本身是 `原cwnd/2`，两步动作易漏算；
> - IPv4分片重组要求**标识相同、片偏移连续、最后一片MF=0**，缺一不可；
> - LRU在周期性访问序列（如1,2,3,4,1,2,3,4…）中可能持续缺页，需结合具体序列推演。

### 5. 下一步备考建议（聚焦具体动作）
1. **机制转计算**：将已理解的机制转化为408典型计算题。例如：
   - 给一段C代码访问二维数组 + Cache参数 → 计算命中率
   - 给页面走向序列 + 物理块数 → 手推LRU/FIFO缺页次数并对比
   - 给TCP初始cwnd/ssthresh + 3个冗余ACK → 写出快恢复后cwnd值
2. **构建对比矩阵**：用表格强制自己输出“触发条件-动作-窗口变化-所属阶段”，如TCP四算法、LRU/FIFO/OPT、顺序表/链表、CSMA/CD vs CSMA/CA。
3. **抓边界与例外**：408最爱考“特例”。例如：
   - UDP检验和在IPv4中可选但强烈建议开启，IPv6强制
   - CSMA/CD仅适用于半双工共享介质，全双工交换机禁用
   - 缺页中断属于异常（Exception），非中断（Interrupt）
